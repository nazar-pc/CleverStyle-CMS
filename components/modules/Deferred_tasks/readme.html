Deferred tasks allows other components to create tasks, that can be executed not immediately, but little bit later. Usually is used for parsers, rating calculators and other purposes, when it is necessary to do some tasks on regular basis.

Tasks addition and deletion.

  • Addition

    \cs\modules\Deferred_tasks\Deferred_tasks::instance()->add($module, $data, $expected, $begin = 0, $priority = 1)

    $module
    Current module, that adds task
    $data
    Any data needed for the task execution
    $expected
    Max time in seconds, during which task is expected to be finished
    $begin
    Unix timestamp, task will not be executed until this time
    $priority
    Priority 0..2, higher number - higher priority
    Task addition is also needed at the end of execution in order to make repeating tasks.

  • \cs\modules\Deferred_tasks\Deferred_tasks::instance()->del($id)

    $id
    Id of task

    Normally task is deleted after execution automatically, but sometimes you may need to delete it earlier, or prevent repeat execution if previous execution failed.

Next trigger is used for task execution:

  • Deferred_tasks/{module_name}

    [
	'id'		=> task_id
	'data'		=> data
]
    {module_name}
    For easy separation - trigger name depends on specified module when task was added
    id
    Task id
    data
    Data, specified during task addition

There are several other methods, but they are used directly not so frequent, so, you can find them with description in source code and IDE suggestions.
